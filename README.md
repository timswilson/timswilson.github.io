## Quick Start (Running Locally)
http://localhost:4000/

```bash
bundle install                    # first time only, installs dependencies
bundle exec jekyll serve          # starts local server at http://localhost:4000/
bundle exec jekyll serve --watch  # same as above, but auto-reloads on file changes
```
> **Note:** Changes to `_config.yml` require you to restart the server.

## Files You Should NOT Touch

| Folder / File | Why |
|---|---|
| `_site/` | Auto-generated by Jekyll on build. Never edit directly — your changes will be overwritten. |
| `_sass/bootstrap/` | Bootstrap source SCSS. Override via `_variables.scss` instead. |
| `Gemfile.lock` | Auto-generated dependency versions. Only changes when you run `bundle install`. |
| `.gitignore` | Already configured, no reason to change. |

## Where to Update Things

### Header / Navigation
**File:** `_includes/header.html`  
Contains the fixed top nav bar (logo + About/Contact links). Update nav link text or add new links here.

### Footer
**File:** `_includes/footer.html`  
Contains the copyright line and any footer links. Update the year and location here.

### Homepage Content
**File:** `index.html`  
This is the main page — hero section, about section, contact section, etc. All of the visible page content lives here.

### TypeIt Animation
**File:** `assets/js/typeIt.js` 
The typing animation on the homepage. Update the `.type()` chains to change what gets typed. See https://typeitjs.com for API docs.

### Page Layout / Structure
**File:** `_layouts/default.html`  
The shell that wraps every page — `<html>`, `<head>`, includes for header/footer, and the `{{ content }}` tag where page content gets injected. You rarely need to touch this unless you're adding a new script or stylesheet globally.

### Site-Wide Config
**File:** `_config.yml`  
Site title, description, URL, and other Jekyll settings. Update this if you change your domain, site title, or social links.

### Head / Meta Tags / Fonts
**File:** `_includes/head.html`  
Everything inside `<head>` — meta tags, font imports (Inter from Google Fonts), CSS links. Add new stylesheets or change the page title format here.

### Styles
**File:** `assets/css/style.scss`  
Your main stylesheet. Custom CSS goes here (nav styles, section styles, etc.).  
**File:** `_sass/_variables.scss`  
Bootstrap variable overrides — colors, fonts, spacing. Change the site's color palette here.

> **The right way to style:** Override Bootstrap variables in `_variables.scss` first. Only write custom CSS in `style.scss` if Bootstrap doesn't have a variable for it.

### Images
**Folder:** `assets/img/`  
Headshot, family photos, logos, etc. Reference them in HTML as `/assets/img/your-file.jpg`.

### Favicon
**File:** `favicon.ico` (root directory)

## Common Tasks

### "I want to add a new section to the homepage"
Add the HTML to `index.html`. Follow the pattern of existing sections (section tag + section-label + divider + content). Add any new styles to `style.scss`.

### "I want to change colors or fonts"
Update `_sass/_variables.scss` for Bootstrap-level changes. For custom component colors (like the nav), update `assets/css/style.scss`.

### "I want to add a new page (e.g. /resume/)"
Create a new file like `resume.html` in the root with Jekyll front matter at the top:
```yaml
---
layout: default
title: Resume
---
```
Then add a link to it in `_includes/header.html`.

## Jekyll Reminders

- `_site/` is generated automatically on build — don't edit it
- Files/folders starting with `_` are processed by Jekyll but not output directly
- Front matter (the `---` block at the top of files) is required for Jekyll to process a file
- Liquid templating: `{{ variable }}` outputs a value, `{% include file.html %}` pulls in a partial

Jekyll/Bootstrap is a Jekyll starter project for GitHub Pages with Bootstrap baked right in.

Jekyll/Bootstrap solves the confusing task of integrating Bootstrap with Jekyll by doing the integration for you - _the right way_ - then guiding you through the simple process of customising Bootstrap - _the right way_.

[Jekyll](https://jekyllrb.com/) is a static website generator that works seamlessly with [GitHub Pages](https://pages.github.com/). Whilst [Bootstrap](https://getbootstrap.com/) is the world's most popular front-end component library. Put this all together and you get a fast, secure and highly customisable static website with free hosting.

## Installation

Before you begin, ensure you've installed [Jekyll](https://jekyllrb.com/). Then:

1. `$ git clone https://github.com/robertlove/jekyll-bootstrap.git`
1. `$ cd jekyll-bootstrap`
1. `$ bundle install`
1. `$ bundle exec jekyll serve`

```bash
bundle install
bundle exec jekyll serve
```

Visit http://localhost:4000/

## Deploying Changes

1. Make your edits in `index.html` (or other files)
2. Commit:
   ```bash
   git add .
   git commit -m "Update site content"
   git push origin master
   ```
3. GitHub Pages auto-deploys in ~1-2 minutes

---

## Tips

- **Use the local server** — Edit locally, check `localhost:4000`, then push
- **Keep `Gemfile.lock` committed** — Ensures consistent Jekyll versions
- **Images** — Put in `assets/img/` and reference as `/assets/img/filename.jpg`
- **Dark mode is baked in** — The current site uses custom CSS, not Bootstrap. Edit `index.html` or `assets/css/style.scss` to change colors

---

## Troubleshooting

**Site not updating after push?**
- Wait 1-2 minutes for GitHub Pages to rebuild
- Check GitHub repo → Actions tab for build errors

**Local server won't start?**
- Run `bundle install` first
- Check Ruby version: `ruby -v`
