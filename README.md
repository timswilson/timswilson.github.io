**GitHub Pages URL:** https://timswilson.github.io/

## Local Development

You need both **Ruby** (via rbenv) and **Node.js** running before you can work locally. Each terminal session requires both servers.

### First-time setup

```bash
# Install Ruby gem dependencies
bundle install

# Install Node.js dependencies (Tailwind CLI)
npm install
```

### Starting the local dev server

You need two terminal windows running simultaneously:

**Terminal 1 — Tailwind (watches for CSS changes):**
```bash
npx tailwindcss -i ./assets/css/input.css -o ./assets/css/style.css --watch
```

**Terminal 2 — Jekyll:**
```bash
bundle exec jekyll serve
```

Site is available at **http://localhost:4000/**

> **Note:** Changes to `_config.yml` require restarting the Jekyll server. Tailwind changes (adding/removing classes in HTML files) are picked up automatically by the `--watch` flag.

## How Deployment Works

Pushing to `master` triggers a GitHub Actions workflow (`.github/workflows/deploy.yml`) that:

1. Installs Node.js dependencies
2. Runs the Tailwind CSS build (`input.css` → `style.css`, minified)
3. Installs Ruby gems
4. Runs `jekyll build`
5. Deploys the `_site/` output to GitHub Pages

**You never push a built `style.css` to the repo** — it's gitignored and always generated fresh in CI.

Changes typically go live within 1–2 minutes of merging to `master`. You can watch the workflow run under the **Actions** tab on GitHub.

## Where to Update Things

### Homepage Content
**File:** `index.html`
Hero section (headshot + TypeIt animation) and contact links. All visible page content lives here.

### TypeIt Animation
**File:** `assets/js/typeIt_Homepage.js`
Controls the typing animation on the homepage. Chain `.type()`, `.pause()`, `.delete()` calls to change what gets typed. Full API: https://typeitjs.com

### Header / Navigation
**File:** `_includes/header.html`
The top nav bar. Add links here if you create new pages.

### Footer
**File:** `_includes/footer.html`
Copyright line and location.

### Styles
**File:** `assets/css/input.css`
Your stylesheet. The top three lines (`@tailwind base/components/utilities`) import Tailwind. Custom component styles (`.contact-link`, `.section-label`, etc.) live in the `@layer components` block below them. Tailwind utility classes go directly in the HTML.

### Page Layout / Structure
**File:** `_layouts/default.html`
The shell wrapping every page — `<head>`, header/footer includes, and `{{ content }}`. Touch this to add global scripts, meta tags, or fonts.

### Site-Wide Config
**File:** `_config.yml`
Site title, description, URL, plugins, and Jekyll settings.

### Images
**Folder:** `assets/img/`
Reference in HTML as `/assets/img/your-file.jpg`.

### Favicon
**File:** `favicon.ico` (root directory)

## Common Tasks

### "I want to style something with Tailwind"
Add utility classes directly in the HTML. Use https://tailwindcss.com/docs as reference. If it's a reusable component style (not a one-off), add it to the `@layer components` block in `assets/css/input.css`.

### "I want to change colors or fonts"
- **Fonts:** The Google Fonts `<link>` is in `_layouts/default.html`. The `font-family` base style is in `assets/css/input.css` under `@layer base`.
- **Colors:** Use Tailwind's built-in color palette (`text-gray-500`, `bg-zinc-100`, etc.) directly in HTML, or define custom colors in `tailwind.config.js` under `theme.extend.colors`.

### "I want to add a new page (e.g. /resume/)"
1. Create `resume.html` in the root with front matter:
```yaml
---
layout: default
title: Resume
---
```
2. Add a link to it in `_includes/header.html`.

### "I want to add a new JS library"
1. Copy the `.js` file to `assets/js/`
2. Add `{% include_relative your-library.js %}` to `assets/js/script.js`
3. Add the filename to the `exclude:` list in `_config.yml` so Jekyll doesn't try to output it as a standalone page

## Files You Should NOT Touch

| File / Folder | Why |
|---|---|
| `_site/` | Auto-generated by Jekyll. Never edit directly. |
| `assets/css/style.css` | Auto-generated by Tailwind. Gitignored — rebuilt on every deploy. |
| `Gemfile.lock` | Auto-generated gem lockfile. Only changes when you run `bundle install`. |
| `package-lock.json` | Auto-generated npm lockfile. Only changes when you run `npm install`. |
| `node_modules/` | npm packages. Gitignored. |

```
_site/                  # Jekyll build output
assets/css/style.css    # Tailwind build output
node_modules/           # npm packages
.idea/                  # JetBrains IDE files
.claude/                # Local Claude Code files
.jekyll-cache/
.bundle/
vendor/
```

## Jekyll Reminders

- Files/folders starting with `_` are processed by Jekyll but not output directly
- Front matter (`---` blocks) is required for Jekyll to process a file with Liquid templating
- `{{ variable }}` outputs a value; `{% include file.html %}` pulls in a partial
- Changes to `_config.yml` require a server restart
